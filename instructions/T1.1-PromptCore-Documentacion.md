# Documentación: Implementación de PromptCore (T1.1)

**Fecha:** 2025-04-23  
**Rama:** `feature/prompt-core`  
**Tarea:** T1.1 - Extraer lógica común de prompts a `src/core/promptCore.ts`

## 1. Resumen

Se ha implementado un módulo central (`PromptCore`) para gestionar todos los prompts del sistema, permitiendo la reutilización de componentes comunes entre diferentes dominios (dental, dermatológico, municipal, educativo, etc.) y facilitando la expansión a nuevos dominios sin duplicar código.

## 2. Archivos creados/modificados

- **Nuevos archivos:**
  - `src/core/promptCore.ts` - Módulo principal con la lógica de prompts
  - `src/core/__tests__/promptCore.test.ts` - Tests unitarios
  - `jest.config.js` - Configuración de Jest para pruebas

- **Archivos modificados:**
  - `src/services/aiservices.ts` - Integración con PromptCore
  - `package.json` - Añadido script de prueba

## 3. Estructura del módulo PromptCore

### 3.1 Tipos principales
```typescript
export type Domain = 'dental' | 'dermatologia' | 'municipal' | 'educacion' | 'eventos' | 'proyectos';

export interface PromptConfig {
  organizationName: string;
  assistantName: string;
  foundationYear?: string;
  location?: string;
  maxEmojisPerMessage?: number;
  domainSpecificConfig?: Record<string, any>;
}
```

### 3.2 Funcionalidades implementadas

- **Carga de prompts por dominio:**
  - Carga dinámica desde archivos en `assets/prompts/prompt_{dominio}.txt`
  - Fallback a prompt genérico si no existe el específico

- **Generación de saludos:**
  - Saludos personalizados con nombre del usuario
  - Variación según hora del día (buenos días/tardes/noches)

- **Mensajes de fallback:**
  - Respuestas predefinidas para errores comunes
  - Mensajes de confirmación genéricos

- **Confirmación de citas:**
  - Generación de mensajes de confirmación parametrizados
  - Soporte para diferentes tipos de servicios y proveedores

- **Composición de prompts:**
  - Sustitución de variables en templates
  - Personalización según contexto

## 4. Integración con AIService

- Inyección de dependencia de PromptCore en AIService
- Configuración inicial con valores por defecto
- Métodos para cambiar de dominio dinámicamente
- Uso de prompts específicos según el dominio actual

## 5. Ejemplo de uso

```typescript
// En un controlador
@inject(AIService) private aiService: AIService;

// Cambiar de dominio según el contexto
aiService.setDomain('municipal');

// El servicio usará automáticamente el prompt adecuado
const respuesta = await aiService.processMessage(mensaje, telefono);
```

## 6. Beneficios

1. **Modularidad:** Separación clara entre lógica común y específica de dominio
2. **Mantenibilidad:** Cambios en componentes comunes se hacen en un solo lugar
3. **Extensibilidad:** Fácil añadir nuevos dominios sin modificar código existente
4. **Consistencia:** Formato uniforme en saludos, mensajes de error y confirmaciones
5. **Testabilidad:** Componentes aislados facilitan pruebas unitarias
