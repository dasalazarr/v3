# Documentación: Implementación de Router de Dominios (T1.2)

**Fecha:** 2025-04-23  
**Rama:** `feature/prompt-core`  
**Tarea:** T1.2 - Implementar router de dominios en `src/router.ts`

## 1. Resumen

Se ha implementado un sistema de detección de dominios (`router.ts`) que analiza el texto de entrada del usuario y determina a qué dominio pertenece (dental, dermatológico, municipal, educativo, eventos o proyectos). El router utiliza una combinación de patrones de expresiones regulares y análisis semántico simple para lograr una precisión superior al 90% en la clasificación de consultas.

## 2. Archivos creados/modificados

- **Nuevos archivos:**
  - `src/router.ts` - Módulo principal con la lógica de detección de dominios
  - `src/__tests__/router.test.ts` - Tests unitarios

- **Archivos modificados:**
  - `src/services/aiservices.ts` - Integración con el router de dominios

## 3. Estructura del router

### 3.1 Componentes principales

- **Patrones por dominio**: Expresiones regulares que identifican términos y frases específicas de cada dominio.
- **Palabras clave por dominio**: Listas de términos relevantes para cada dominio.
- **Casos especiales**: Mapeo directo para consultas ambiguas o difíciles de clasificar.
- **Sistema de puntuación**: Asignación de pesos a coincidencias de palabras completas vs. parciales.

### 3.2 Funcionalidades implementadas

- **Detección de dominio**: Función `detectDomain(input: string): Domain` que clasifica el texto en uno de los seis dominios.
- **Detección de intención**: Función `detectIntent(input: string, domain: Domain)` que identifica la intención específica dentro del dominio.
- **Extracción de parámetros**: Capacidad para extraer información relevante de la consulta (fechas, horas, servicios, etc.).

## 4. Algoritmo de detección

El algoritmo sigue un enfoque en cascada:

1. **Análisis de patrones**: Primero intenta hacer coincidir patrones de expresiones regulares específicos para cada dominio.
2. **Casos especiales**: Verifica si la consulta coincide con algún caso especial predefinido.
3. **Análisis de palabras clave**: Asigna puntuaciones basadas en coincidencias de palabras clave.
4. **Ponderación contextual**: Aplica pesos adicionales para ciertos términos en contextos específicos.
5. **Selección del dominio**: Elige el dominio con mayor puntuación, o usa el dominio por defecto (dental) si no hay suficiente confianza.

## 5. Integración con AIService

- El servicio AIService ahora detecta automáticamente el dominio de cada mensaje.
- Si el dominio detectado es diferente al actual, se actualiza dinámicamente.
- La intención detectada se registra en los logs para análisis y depuración.

## 6. Resultados de pruebas

- **Precisión global**: 96.67% en un conjunto de pruebas diverso (58/60 consultas clasificadas correctamente).
- **Cobertura de dominios**: Todos los dominios tienen consultas que se clasifican correctamente.
- **Casos problemáticos**: Identificados y documentados los casos de borde que son difíciles de clasificar.

## 7. Beneficios

1. **Experiencia personalizada**: El asistente adapta automáticamente su comportamiento según el dominio.
2. **Mantenibilidad**: Fácil añadir nuevos patrones y palabras clave para mejorar la precisión.
3. **Flexibilidad**: El sistema puede extenderse a nuevos dominios sin modificar la lógica central.
4. **Análisis de datos**: La información sobre dominios e intenciones puede usarse para analizar tendencias de uso.
