# Documentación: Integración del Router con Flujo de Conversación (T1.3)

**Fecha:** 2025-04-23  
**Rama:** `feature/prompt-core`  
**Tarea:** T1.3 - Integrar router con flujo de conversación en `src/services/conversacion.controller.ts`

## 1. Resumen

Se ha implementado un controlador de conversación (`ConversacionController`) que integra el router de dominios con el flujo de conversación del asistente. Este controlador actúa como intermediario entre las solicitudes del usuario y el servicio de IA, detectando automáticamente el dominio y la intención de cada mensaje para proporcionar respuestas contextualizadas.

## 2. Archivos creados/modificados

- **Nuevos archivos:**
  - `src/services/conversacion.controller.ts` - Controlador principal para gestionar conversaciones
  - `src/services/__tests__/conversacion.controller.test.ts` - Tests de integración

## 3. Estructura del Controlador

### 3.1 Componentes principales

- **Interfaz `ConversationResponse`**: Define la estructura estándar de respuestas del controlador, incluyendo metadatos sobre el dominio e intención detectados.
- **Clase `ConversacionController`**: Implementa la lógica para procesar mensajes, iniciar conversaciones y cambiar de dominio.

### 3.2 Funcionalidades implementadas

- **Procesamiento de mensajes**: Método `processMessage` que integra la detección de dominio e intención con el procesamiento de mensajes del servicio AI.
- **Inicio de conversación**: Método `startConversation` que genera mensajes de bienvenida personalizados según el dominio.
- **Cambio de dominio**: Método `changeDomain` que permite cambiar explícitamente el contexto de la conversación.
- **Mensajes contextualizados**: Generación de respuestas adaptadas al dominio detectado.

## 4. Flujo de Procesamiento

El controlador sigue el siguiente flujo para cada mensaje:

1. **Detección de dominio**: Utiliza el router para determinar a qué dominio pertenece el mensaje.
2. **Detección de intención**: Identifica la intención específica dentro del dominio.
3. **Actualización de contexto**: Si el dominio detectado es diferente al actual, lo actualiza.
4. **Procesamiento del mensaje**: Envía el mensaje al servicio AI para generar una respuesta.
5. **Estructuración de la respuesta**: Devuelve una respuesta enriquecida con metadatos sobre el dominio, intención y parámetros detectados.

## 5. Integración con AIService

- El controlador utiliza el servicio AIService para procesar los mensajes y mantener el contexto de la conversación.
- Gestiona automáticamente los cambios de dominio basados en la detección del router.
- Proporciona una capa de abstracción que simplifica la interacción con el asistente desde otros componentes del sistema.

## 6. Ejemplos de Uso

### Procesamiento de un mensaje
```typescript
const controller = container.resolve(ConversacionController);
const response = await controller.processMessage(
  "¿Cuánto cuesta un blanqueamiento dental?",
  "+1234567890"
);

// response contiene:
// - message: Respuesta del asistente
// - domain: "dental"
// - intent: "consultar_precio"
// - params: { servicio: "blanqueamiento" }
// - metadata: Información adicional sobre la conversación
```

### Inicio de una conversación
```typescript
const response = await controller.startConversation(
  "+1234567890",
  "educacion" // Dominio inicial opcional
);

// Devuelve un mensaje de bienvenida contextualizado
```

## 7. Beneficios

1. **Experiencia coherente**: Mantiene el contexto de la conversación a lo largo de múltiples mensajes.
2. **Respuestas contextualizadas**: Adapta las respuestas según el dominio e intención detectados.
3. **Transiciones fluidas**: Gestiona automáticamente los cambios de dominio durante la conversación.
4. **Metadatos enriquecidos**: Proporciona información detallada sobre cada interacción para análisis y depuración.
5. **Manejo de errores robusto**: Captura y gestiona errores de forma elegante para mantener la conversación activa.
