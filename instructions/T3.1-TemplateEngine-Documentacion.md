# Documentación: Integración de Motor de Plantillas (T3.1)

**Fecha:** 2025-04-24  
**Rama:** `feature/template-engine`  
**Tarea:** T3.1 - Integrar engine de plantillas (Handlebars) en `src/core/templateEngine.ts`

## 1. Resumen

Se ha implementado un motor de plantillas basado en Handlebars que permite la generación dinámica de mensajes con variables como `{domain}`, `{user}`, `{nextStep}`, etc. Este motor facilita la creación y mantenimiento de mensajes consistentes en todos los flujos de conversación y dominios del asistente.

## 2. Archivos creados/modificados

- **Nuevos archivos:**
  - `src/core/templateEngine.ts` - Implementación del motor de plantillas
  - `src/core/__tests__/templateEngine.test.ts` - Tests unitarios
  - `src/templates/flows/*.hbs` - Plantillas de mensajes para los diferentes flujos

- **Archivos modificados:**
  - `src/templates/mainflow.ts` - Actualizado para usar plantillas
  - `src/templates/faqFlow.ts` - Actualizado para usar plantillas
  - `src/templates/appointmentFlow.ts` - Actualizado para usar plantillas

## 3. Estructura del Motor de Plantillas

### 3.1 Componentes principales

- **Clase `TemplateEngine`**: Implementa la lógica para cargar, compilar y renderizar plantillas Handlebars.
- **Interfaz `TemplateVars`**: Define la estructura de variables que pueden utilizarse en las plantillas.
- **Helpers personalizados**: Funciones auxiliares como `formatDate` y `when` para extender la funcionalidad de las plantillas.

### 3.2 Funcionalidades implementadas

- **Carga dinámica de plantillas**: Las plantillas se cargan desde archivos `.hbs` en el directorio `src/templates/flows/`.
- **Caché de plantillas**: Las plantillas compiladas se almacenan en memoria para mejorar el rendimiento.
- **Renderizado con variables**: Soporte para variables de contexto como `domain`, `user`, `date`, etc.
- **Helpers personalizados**: Funciones auxiliares para formateo de fechas y condicionales avanzados.
- **Manejo de errores**: Gestión robusta de errores con mensajes de fallback.

## 4. Uso del Motor de Plantillas

### 4.1 Renderizado de plantillas desde archivos

```typescript
import { TemplateEngine } from '../core/templateEngine';

// Obtener instancia (o usar inyección de dependencias)
const templateEngine = new TemplateEngine();

// Renderizar plantilla con variables
const message = templateEngine.render('welcome', {
  user: { name: 'Juan' },
  domain: 'dental'
});
```

### 4.2 Renderizado de plantillas desde strings

```typescript
const template = 'Hola {{user.name}}, bienvenido al dominio {{domain}}.';
const message = templateEngine.renderString(template, {
  user: { name: 'María' },
  domain: 'dermatologia'
});
```

### 4.3 Uso de helpers

```handlebars
{{!-- Formateo de fechas --}}
Tu cita es el {{formatDate date 'full'}} a las {{formatDate time 'time'}}

{{!-- Condicionales avanzados --}}
{{#when domain 'eq' 'dental'}}
  Contenido específico para dominio dental
{{else}}
  Contenido para otros dominios
{{/when}}
```

## 5. Plantillas Implementadas

Se han creado plantillas para los siguientes flujos:

- **Bienvenida**: Saludo inicial personalizado según el dominio.
- **Citas**: Solicitud de fecha/hora, confirmación, resumen y éxito/error.
- **Cancelación de citas**: Solicitud de ID, confirmación y éxito/error.
- **Errores**: Mensajes de error genéricos y específicos.

## 6. Integración con Flujos Existentes

Los flujos de conversación existentes (`mainflow.ts`, `faqFlow.ts`, `appointmentFlow.ts`) han sido actualizados para utilizar el motor de plantillas, reemplazando los mensajes estáticos por llamadas a `templateEngine.render()`.

## 7. Beneficios

1. **Consistencia**: Mensajes uniformes en todos los flujos y dominios.
2. **Mantenibilidad**: Separación de la lógica de presentación y la lógica de negocio.
3. **Flexibilidad**: Fácil adaptación a diferentes dominios y contextos.
4. **Internacionalización**: Base para futura implementación de múltiples idiomas.
5. **Personalización**: Mensajes adaptados al usuario y su contexto.
